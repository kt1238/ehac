Activate some dead code

You need to find the password for remotely disarming all atomic weapons of Ruritania (that's coincidentally your flag). A smart computer scientist has hidden such password into some dead code in a binary executable file. However, the binary file suffers from buffer overflow in its input. How to outsmart the smart computer scientist? To do that you need to find a suitable input to the vulnerable binary file that runs the piece of dead code which will reveal the hidden piece of information.
Some commands which can be useful (or not):

  • objdump -d <binary-file> disassembles the whole content of an executable.
  • perl -e 'print "A" x 10' prints the character 'A' for 10 times.
  • perl -e 'print "A" x 10 . "BBBB" ' prints the character 'A' for 10 times followed by 4 Bs.
  • gdb <binary-file> starts in the GNU Debugger on an executable. In this environment, you can type the following commands:
    • run to starts the program.
    • break <address> to introduce a break-point at the address, e.g. *0x08048441.
    • cont to continue after a breakpoint.
    • x/16x $ebp to print out 16 bytes stored at the address pointed by the frame pointer.
